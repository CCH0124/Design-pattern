# 享元模式 (Flyweight Pattern)

在處理大規模分散式系統、遊戲伺服器開發，或是需要處理海量資料的場景時，我們經常面臨「記憶體耗盡 (OOM)」或系統效能低下的問題。如果系統需要產生成千上萬個極為相似的物件，**享元模式 (Flyweight Pattern)** 就是我們節省記憶體與優化效能的終極武器。

1. 享元模式的核心概念

   **定義：** 當一個類別的單一實例（Instance），可以用來提供許多個「虛擬」的實例時，就可以使用享元模式。

   享元模式的核心精神在於**「資源共用」**。它透過將物件的狀態分為可以共用的「內部狀態 (Intrinsic)」與不可共用、需外部傳入的「外部狀態 (Extrinsic)」，來大幅減少執行時期 (Runtime) 需要建立的物件數量，進而節省極大的記憶體空間。

2. 系統問題與解決方案

   讓我們用一個書中的場景來說明：假設你正在開發一款景觀設計應用程式，使用者會在畫面上種植成千上萬棵「樹 (Tree)」。

   **傳統作法的災難：**
        如果採用傳統的物件導向寫法，每一棵樹都是一個獨立的物件，各自在記憶體中維持自己的 `x` 座標、`y` 座標以及 `age` (樹齡)。當你的大客戶在大型社區建案中建立了一大片樹林（例如百萬棵樹）時，龐大的物件實例會塞爆系統的 Heap 記憶體，導致應用程式變得非常遲鈍。

   **享元模式的優化：**
        我們將樹的狀態拆分。把每一棵樹獨有的資料（如 `x`, `y`, `age`）抽離出來，集中在一個「樹木管理器 (TreeManager)」的二維陣列中統一維護。而在系統中，我們**永遠只建立「一個」沒有狀態 (State-free) 的 Tree 實例**。

   當管理器需要畫出這片樹林時，它會遍歷陣列，並把該樹專屬的 `x`, `y`, `age` 當作參數，傳遞給那唯一一個 Tree 物件去執行繪圖動作。

   流程圖輔助理解:

   ```text
   [ 傳統做法：災難性的記憶體消耗 ]
   [Tree 1: x,y,age] \
   [Tree 2: x,y,age] - 皆存在獨立記憶體中 (產生 1,000,000 個物件，耗費海量記憶體)
   ...               /
   [Tree 1000000]   /
   
   ========================================================================
   
   [ 享元模式 (Flyweight)：效能優化架構 ]
   
   (1) 集中管理外部狀態 (Extrinsic State)
   [ TreeManager (樹木管理器) ]
   + treeArray: 統一維護所有虛擬樹的 [x, y, age] 資料 
   + displayTrees() -------------------+
                                       |
                                       v (2) 迴圈遍歷，將 [x,y,age] 當作參數傳遞給享元物件
                                       |
   (3) 唯一且無狀態的實例 (Flyweight)  |
   [ 單一 Tree 物件 ] <----------------+ (系統中只有 1 個物件)
   + display(x, y, age) : 接收外部參數，並負責真正的繪圖邏輯
   ```

3. 總結

   對於系統工程師而言，架構設計永遠是時間與空間的權衡 (Trade-off)。享元模式其實就是用「運算時間 (傳遞參數)」來換取「記憶體空間」。

   **主要優點 (Benefits)：**
    * **極致的記憶體優化：** 透過減少執行時期的物件實例數量，大幅降低記憶體消耗。
    * **狀態集中管理：** 將許多「虛擬」物件的狀態集中到單一位置 (如 Manager) 進行維護。
    * **適用於海量相似物件：** 當一個類別有非常多實例，且它們的行為與控制方式幾乎一致時，這是最完美的解決方案。

   **主要缺點與代價 (Drawbacks)：**
    * **失去物件的獨立行為：** 這是實作享元模式最大的妥協。因為底層共用同一個邏輯實例，這些虛擬出來的物件將無法再像獨立物件那樣，擁有自己獨特的行為或自我演化。
    * **增加系統複雜度：** 工必須小心翼翼地切分內部狀態與外部狀態，並額外設計管理器來維護這些外部狀態。 

   在核心 Java 中的示例：
     * java.lang.Integer#valueOf(int) （以及 Boolean、 Byte、 Character、 Short、 Long 和 BigDecimal）
